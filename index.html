Project Overview

The Problem

Excessive plant growth shades solar panels and cuts energy output. Keeping rows clear is essential — yet manual vegetation assessment across hundreds of acres is slow, uneven, and expensive. Mowing is among the highest recurring costs in solar operations.

  • Time-consuming, labor-heavy walk or drive surveys
  • Inconsistent coverage across very large sites
  • Repeated visits and crew time drive up mowing budgets

Our Solution

NextVision (Team 3) is the vehicle rig: dual cameras, GPS, and edge storage. AgriSight (Team 7) ingests those images in AWS, runs ResNet-50 classification, and serves maps and field scores — cutting mowing cost and manual survey labor through data-driven maintenance planning.

  • Fleet trucks become survey vehicles — no dedicated inspection fleet required
  • GPS-tagged captures synced to AWS for ResNet-50 inference
  • Dashboard maps + field score for crew dispatch and weekly reporting

System Architecture

NextVision (Team 3) on the truck → NVMe & S3 → SageMaker → DynamoDB → AgriSight (Team 7) in the browser — built for NextEra-scale solar fields.

NextVision is the EECE404 Team 3 hardware platform — everything mounted on the vehicle before the cloud.

  • Imaging: 2× global-shutter MIPI cameras — distortion-free, synchronized capture
  • Compute: Raspberry Pi 5 (dual-stream capture + headroom)
  • Storage: 500GB NVMe SSD via PCIe HAT — offline buffer when network is unavailable
  • GPS: u-blox NEO-M8T — sub-meter geo-tags on every image
  • Power: LiFePO₄ 12.8V / 50Ah + DC-DC → stable 5V for Pi (XT60 + harness)
  • Environmental: IP67 enclosure, cable glands, anti-vibration pads
  • Mounting: Stainless hardware + aluminum bracket — F-250 truck rail

AgriSight is the EECE404 Team 7 software stack: inference, APIs, data store integration, and this website.

Cloud inference (AWS SageMaker)

ResNet-50 classifier (transfer learning) on field-collected imagery. Ingestion via REST / JSON aligned with enterprise GIS and reporting tools.

Classes: no_vegetation — panel surface clear; vegetation — plant coverage on/near panel; other — non-target / robustness.

AgriSight web dashboard (Team 7)

This site: HTML, Tailwind CSS, and vanilla JavaScript with the Google Maps JavaScript API — filters, Eastern Time timestamps, camera ID, and a field score derived from loaded detections.

Privacy & compliance (design intent)

GDPR-aligned handling of imagery; geofencing to disable capture in sensitive zones; NERC CIP–style secured AWS paths; truck-mounted operation only (no UAS).

End-to-end flow

Dual capture → NVMe logging → automated S3 sync on Wi-Fi → SageMaker inference → records in DynamoDB → AgriSight (Team 7) map & API.

Immutable audit trail

Each classification stores GPS coordinates, camera ID, S3 object path, and ML model version — supporting compliance, validation, and retraining.

Decision loop (operations)

Weekly-style field scores from vegetation density in the data; threshold breaches highlight map pins so crews can target mowing and trimming.

Model training and validation are led by AgriSight (EECE404 Team 7), using imagery from NextVision (Team 3) deployments.

ResNet-50 on AWS SageMaker — chosen for residual blocks (stable training vs. very deep CNNs), strong accuracy-to-parameter ratio, and transfer learning on a specialized dataset (938 labeled field images).

Validation snapshot

  • no_vegetation — strong F1 (~0.94); majority class in validation
  • other — F1 ~0.91
  • vegetation — F1 ~0.57, recall ~0.55; small training/val count — priority for more labeled vegetation samples

Class imbalance (few vegetation examples vs. clear panels) is the main limiter; expanding vegetation diversity in the dataset is the next accuracy lever.

Outcomes & impact

NextVision (Team 3) feeds the field; AgriSight (Team 7) closes the loop to crew dispatch — lowering mowing spend and manual survey burden for utility-scale solar.

End-to-end pipeline

Images to S3, SageMaker inference, DynamoDB records — automated with no manual upload step in the loop.

ResNet-50 in production path

Audit-ready records

GPS, camera ID, S3 path, and model version on every row — traceable for compliance and retraining.

Actionable dashboard

GIS map plus field score translate ML output into maintenance priorities for the field org.
